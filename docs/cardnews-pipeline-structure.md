# 카드뉴스 파이프라인 — 시드 기획 + 카드 기획 중심

> **작성:** 2026-05-19
> **채널:** @mamurs.ai.lab
> **핵심 전환:** 페이지 생성은 완성됨 → 이제 **시드 기획 + 카드 기획**에 모든 자원 집중

---

## 0. 패러다임 전환

### 0.1 무엇이 바뀌었나
| | 이전 작업 (잘못) | 이제부터 |
|---|---|---|
| 시드 결정 | 제가 임의로 ("회의록", "Skills") | **검증된 데이터에서 자동 도출** |
| 카드 기획 | 8장 카피만 잘 쓰면 됨 | **각도·관점·evidence 채택이 핵심** |
| 페이지 생성 | 매번 새로 검증 | **이미 완성된 라인 — 그대로 사용** |

### 0.2 작업 비중 재분배
```
이전:  카피 50% / 이미지 40% / 시드 10%
이제:  시드 40% / 카드 50% / 이미지 10% (자동)
```

→ 시드·카드 기획이 잘 되면 페이지 생성은 자동 끝남.

## 1. Stage 1 — 시드 기획 (Seed Planning)

### 1.1 목적
"AI" 같은 입력 키워드 → **viral 가능성 검증된 시드 1개**까지 좁히기

### 1.2 시드 후보 자동 생성 (5~10개)

**입력:** 키워드 1개 (예: "ai", "자동화", "직장인")

**출력:** 시드 후보 카드 5~10개, 각 카드에 다음 데이터 포함

| 필드 | 예시 | 출처 |
|------|------|------|
| 시드명 | "Claude Skills 만들기" | LLM 확장 |
| **빈도** | DB 266건 / Reddit 58↑ | 실데이터 |
| **화제성** | "2026-05 hot" / "1주 internal 검색 ↑127%" | 트렌드 지표 |
| **차별성** | "한국 후기 X / 영문 공식만 있음" | 정보비대칭 점수 |
| **evidence 풍부도** | 도구명 3개+ / 수치 2개+ / 사례 5개+ | 검증 자동 |
| **4-Gate 예측 점수** | Why now ✅ / Why us ✅ / So what ✅ / Save ⚠️ | 자동 평가 |

### 1.3 시드 선정 룰
- evidence 풍부도 ★★★ (도구·수치·사례 모두 있어야)
- 4-Gate 4개 중 3개 이상 ✅
- 차별성 ★★ (한국어 후기 없거나 적은 신선한 주제)
- 첫 편(회의록)·2편(Skills)와 메시지 중복 X

### 1.4 시드 자동 평가 알고리즘 (제안)
```
score(시드) =
  (DB 빈도 정규화 × 0.2)
  + (화제성 시의성 × 0.25)
  + (evidence 풍부도 × 0.30)
  + (정보비대칭 점수 × 0.15)
  + (시리즈 차별성 × 0.10)
```

상위 5~10개를 시드 후보 카드로 노출, 사용자가 1개 선택.

### 1.5 ✅ 시드 기획 완료 기준
- 시드 1개 확정
- 4-Gate 4개 통과
- evidence 3개+ 검증 완료
- 첫·2편과 차별점 명확

## 2. Stage 2 — 카드 기획 (Card Planning)

### 2.1 목적
선정된 시드 1개 → **8장 카피·evidence·수치·각도** 완성

### 2.2 카드 기획 5단계

#### Step ①. 각도 설정
어떤 관점에서 시드를 풀 것인가? 4트랙 중 선택:

| 트랙 | 패턴 | 사례 |
|------|------|------|
| **A. 실행 가이드** | "방법 안 알려주는 X / 이렇게 하시면 돼요" | 회의록 자동화 |
| **B. 신기능 활용** | "신기능 나왔는데 / 진짜 써봤더니" | Claude Skills |
| **C. 비교·검증** | "5개 1주씩 써봤어요 / 진짜는 1개" | 미정 (다음 편 후보) |
| **D. 실패담** | "이거 하지 마요 / 30개 깔고 26개 삭제" | 미정 |

#### Step ②. P1 후크 결정
- 충격 수치 1개 (P7과 연결)
- 주제 명시 (단어 노출)
- 약속 톤 ("이렇게 하시면 돼요" 식)
- 10자 이내 2줄 구조

#### Step ③. evidence 채택 (저장 가치 핵심)
- 도구 이름 2~4개 (실제 존재·검증)
- 수치 1~2개 (실제 후기 출처)
- 사례 1~2개 (한국 직장인 매칭)
- → P4·P5·P6·P7에 분배

#### Step ④. 8장 흐름 매핑

| P | 역할 | 채울 것 |
|---|------|---------|
| P1 | 후크 | 충격 수치 + 주제 + 약속 |
| P2 | 정보비대칭 | 시장 통념·문제 자각 |
| P3 | 1인칭 노력 | "그래서 한 달 해봤거든요" |
| P4 | 1단계 | 첫 행동 (체크리스트 또는 도구) |
| P5 | 2단계 | 핵심 도구 + 사용법 |
| P6 | 3단계 | 결정적 디테일 (프롬프트·규칙) |
| P7 | 결과 | 수치 변화 (실제 후기 매칭) |
| P8 | 저장 CTA | 다음 주 행동 1개 |

#### Step ⑤. 자가 검증
- [ ] playbook 16룰 통과
- [ ] 4-Gate 통과
- [ ] evidence 출처 명시
- [ ] P3 1인칭 voice
- [ ] P7 수치가 P1과 연결
- [ ] P8 CTA 행동 명확

### 2.3 카드 기획 산출물
JSON 1개로 정리:
```json
{
  "seed": "Claude Skills 만들기",
  "angle": "B. 신기능 활용",
  "evidence": {
    "tools": ["Claude.ai", "skill-creator", "SKILL.md"],
    "numbers": ["30분→3분", "1시간→10분"],
    "cases": ["1인 마케터", "쭈로로 매달 보고서"]
  },
  "cards": [
    { "p": "P1", "l1": "매주 보고서 30분", "l2": "3분 됐어요", "accent": "3분" },
    { "p": "P2", "l1": "...", "l2": "...", "accent": "..." },
    ...
  ]
}
```

→ 이 JSON이 **페이지 생성기에 그대로 입력**되면 8장 이미지가 자동.

## 3. Stage 3 — 페이지 생성 (이미 완성)

### 3.1 입력
Stage 2의 JSON 1개

### 3.2 자동 처리
- P1: 사진 배경 + 헤드라인 burn-in
- P2~P8: 다크 그라데이션 + 상단 일러스트 + 텍스트
- accent 박스 자동 적용
- 핸들 좌하단 자동
- 페이지번호·부연·메타라벨 ❌ (자동 제거)

### 3.3 출력
8장 이미지 (9:16, 1080x1920)

### 3.4 ✅ 이미 검증된 spec
- 다크 네이비 `#0A1F2E` → 다크 그린 `#0E2A24` 그라데이션
- 보라 `#6C63FF` + 민트 `#00D4AA` accent
- 흰색 `#F5F5F5` 헤드라인 / `#C7CED8` 본문
- Pretendard Black 폰트
- 한글 받침 정확
- (상세는 이전 검증된 spec — 본 문서에서 다시 안 다룸)

## 4. Contents 페이지 (`nowlink.kr/contents`) 재구성안

### 4.1 현재 구조 진단
현재 4-STEP UI:
- STEP 1: 트렌드 수집 (키워드 → YT/Reddit/Clien)
- STEP 2: 인사이트 도출 (AI 추출 + 본인 인사이트)
- STEP 3: 주제·주장 분리
- STEP 4: 카드뉴스 생성

→ 사용자가 임의 키워드 치고, AI가 인사이트 만들고, 카드뉴스가 만들어짐.
→ **시드 검증·카드 기획 흐름이 약함** (사용자가 너무 많은 결정을 매번 함)

### 4.2 재구성 목표
**클릭 3번으로 시드 → 카드 → 이미지 완성**

| 새 STEP | UI | 자동/수동 |
|---------|----|---------|
| **STEP 1** 시드 발굴 | 키워드 1개 입력 → 시드 후보 5~10개 카드 자동 | 자동 |
| **STEP 2** 시드 선정 | 후보 카드 클릭 1번 → 4-Gate 점수 자동 표시 → "이걸로" 버튼 | 1클릭 |
| **STEP 3** 카드 기획 | 각도 1개 선택 (A/B/C/D 트랙) → evidence·8장 카피 자동 생성 → 인라인 수정 | 1선택 + 미세 수정 |
| **STEP 4** 페이지 생성 | "생성" 버튼 → 8장 이미지 자동 | 자동 |

### 4.3 제거할 것 (UI 간소화)
- ❌ "시드 단축 입력" / 6개 분야 탭 — 시드 발굴이 자동이라 불필요
- ❌ "본인 인사이트" 수동 입력칸 — 카드 기획 단계로 흡수
- ❌ "주제·주장 분리" 별도 STEP — 카드 기획에 자동 포함
- ❌ "전체 JSON" 노출 — 디버그용으로 숨김
- ❌ "에이전트 로그" 메인 영역 차지 — 사이드패널/접힘

### 4.4 강조할 것
- ✅ 시드 후보 카드의 **데이터 시각화** (빈도·화제성·4-Gate)
- ✅ 시드 선정 후 **각도 선택 4트랙** (큰 버튼 4개)
- ✅ 8장 카피 **인라인 수정 가능** (각 P별 한 줄 편집)
- ✅ "이전 편과 차별점" 표시 (시리즈 일관성 보장)

### 4.5 액션 리스트 (실제 코드 작업)

| 우선순위 | 작업 | 영향 파일 |
|---------|------|----------|
| 🔴 P1 | STEP 1을 "시드 발굴"로 재정의 | `public/contents-app/index.html` 줄 ~?? (STEP 1 영역) |
| 🔴 P1 | 시드 후보 카드 UI (5~10개 그리드) | 신규 컴포넌트 |
| 🔴 P1 | 시드 카드에 데이터 표시 (빈도/화제성/4-Gate) | `api/contents/seed-options/route.ts` 확장 |
| 🟡 P2 | STEP 2를 "시드 선정 + 4-Gate 점수"로 | UI 재배치 |
| 🟡 P2 | STEP 3을 "각도 4트랙 + 8장 카피 자동"으로 | `api/contents/insights/propose/route.ts` 활용 |
| 🟢 P3 | STEP 4 "페이지 생성" — 이미 작동 ✅ | (변경 없음) |
| 🟢 P3 | 좌측 보조 메뉴 (이전 편 일관성 체크) | UI 추가 |

### 4.6 작업 안전성
- 4,000줄 HTML 한 번에 손대지 말 것
- STEP 1부터 1개씩 변경 + 매번 검증
- 기존 API는 유지 (UI만 재배치)

## 6. 결론 — 이제부터의 작업 흐름

### 6.1 매주 1편 발행 flow (목표)
```
[월요일]
1. nowlink.kr/contents에 입력 키워드 1개
2. 시드 후보 5~10개 자동 노출 (Stage 1)
3. 그 중 1개 선정 + 4-Gate 점수 확인 (1클릭)

[화요일]
4. 각도 4트랙 중 1개 선택 (A/B/C/D)
5. 8장 카피 자동 생성 + 인라인 수정 (Stage 2)

[수요일]
6. "생성" 버튼 → 8장 이미지 자동 (Stage 3 — 이미 완성)

[목요일]
7. 인스타 업로드

[금요일~일요일]
8. 저장·댓글·팔로워 지표 측정
```

### 6.2 측정 지표 (편당)
- 저장 수 / 좋아요 비율
- P1~P8 중 가장 머문 페이지
- 댓글 키워드 (어떤 단계에 반응)
- 프로필 방문 → 팔로워 전환률
- 이걸 다시 Stage 1 시드 평가 알고리즘에 피드백

### 6.3 핵심 메시지
> **페이지 생성은 끝났다. 이제 시드·카드 기획만 잘하면 viral.**
